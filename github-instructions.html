GitHub Update Instructions

This document provides instructions on how to update the Team Blitz repository on GitHub with the latest changes (version 0.2.0).

Note: These instructions assume you have Git installed on your computer and have access to the Team Blitz GitHub repository.

What Has Been Updated

The following files have been updated:

  • README.md - Improved design with badges, team member profiles, and better formatting
  • package.json - Version updated from 0.1.0 to 0.2.0

Step-by-Step Instructions

Step 1: Open a Terminal or Command Prompt

Open a terminal or command prompt and navigate to the project directory:

cd path/to/Team-Blitz/hackathon-team-landing

Step 2: Initialize Git Repository (if not already done)

If the project is not already a Git repository, initialize it:

git init

Step 3: Configure Git (if not already done)

Set your Git username and email if not already configured:

git config user.name "Aditya Kumar Tiwari"
git config user.email "itisaddy7@gmail.com"

Step 4: Add Files to Git

Add the modified files to Git:

git add README.md package.json

Step 5: Commit Changes

Commit the changes with a descriptive message:

git commit -m "Update to version 0.2.0 with improved README design"

Step 6: Add Remote Repository (if not already done)

If you haven't already connected to the GitHub repository, add it as a remote:

git remote add origin https://github.com/Xenonesis/Team-Blitz.git

Step 7: Push Changes to GitHub

Push the changes to GitHub:

git push -u origin master

If you're using a different branch, replace "master" with your branch name.

Authentication: You may be prompted to enter your GitHub username and password or a personal access token. If you have two-factor authentication enabled, you'll need to use a personal access token instead of your password.

Verifying the Update

After pushing the changes, visit the GitHub repository at https://github.com/Xenonesis/Team-Blitz to verify that the changes have been successfully uploaded.

Troubleshooting

  • If you encounter permission issues, make sure you have write access to the repository.
  • If you get an error about the remote repository not existing, double-check the repository URL.
  • If you have conflicts, you may need to pull the latest changes first with git pull origin master.
